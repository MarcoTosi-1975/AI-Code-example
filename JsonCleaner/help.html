📦 JsonCleaner.dll

Libreria .NET per la rimozione ricorsiva di elementi JSON

Compatibile con TestStand 2019+ | 🆕 Nuova Sintassi Semplificata!

📋 Panoramica

JsonCleaner è una libreria .NET che permette di rimuovere elementi da file JSON in modo ricorsivo, indipendentemente dal livello di annidamento.

🎉 Novità Versione 1.1:
  • Sintassi filtri semplificata - Zero escape, zero virgolette complicate!
  • Scrivi semplicemente: "Occurred Equals true"
  • Niente più: "[{\"Property\": \"Occurred\", ...}]"

Caratteristiche Principali

  • ✅ Rimozione ricorsiva di oggetti, array e proprietà
  • ✅ Sistema di filtri condizionali con sintassi semplificata 🆕
  • ✅ Logging dettagliato opzionale
  • ✅ Gestione memoria ottimizzata con IDisposable
  • ✅ Protezione stack overflow (max 100 livelli)
  • ✅ Compatibile con TestStand .NET Adapter
  • ✅ Thread-safe e performante

Requisiti

Componente Versione Minima
.NET Framework / .NET 4.8 o .NET 6.0+
Newtonsoft.Json 13.0.3 o superiore
TestStand (opzionale) 2019 o superiore

💾 Installazione

File Necessari

📁 File da copiare sul sistema target:
  • JsonCleaner.dll - Libreria principale
  • Newtonsoft.Json.dll - Dipendenza JSON parser

Posizione Raccomandata

C:\TestStand\Components\JsonCleaner\ ├── JsonCleaner.dll ├── Newtonsoft.Json.dll └── JsonCleaner_Documentation.html

Verifica Installazione

  1. Copiare i file nella cartella scelta
  2. Aprire TestStand Sequence Editor
  3. Creare un nuovo step con .NET Adapter
  4. Selezionare JsonCleaner.dll come Assembly
  5. Verificare che appaia la classe JsonCleaner.JsonCleanerTool

🔧 Metodi Disponibili

1. RemoveJsonElementFromString

Scopo: Rimuove elementi JSON in base al contenuto/valore

Parametri:

Nome Tipo Direzione Descrizione
inputJson String Input Stringa JSON completa da processare
targetToRemove String Input Elemento JSON da rimuovere (oggetto, array, valore)
outputJson String Output JSON ripulito (risultato)
removedCount Int32 Output Numero di elementi rimossi
logOutput String Output Log dettagliato operazioni
enableLogging Boolean Input Optional Abilita logging (default: false)
Return Value Boolean Output True se operazione riuscita

2. RemovePropertyByName 🆕 Aggiornato v1.1

Scopo: Rimuove proprietà JSON in base al nome, con supporto filtri condizionali

Parametri:

Nome Tipo Direzione Descrizione
inputJson String Input Stringa JSON completa
propertyName String Input Nome della proprietà da rimuovere (es. "Error", "Common")
outputJson String Output JSON ripulito
removedCount Int32 Output Numero di proprietà rimosse
skippedCount Int32 Output Numero di proprietà skippate (filtri)
logOutput String Output Log dettagliato
filtersJson String Input Optional JSON array di filtri (vecchio formato)
filtersString 🆕 Nuovo String Input Optional Filtri sintassi semplificata (es. "Occurred Equals true")
enableLogging Boolean Input Optional Abilita logging
Return Value Boolean Output True se successo

3. RemoveJsonElement

Scopo: Come RemoveJsonElementFromString ma opera su file fisici

Parametri identici al metodo 1, ma usa inputFilePath e outputFilePath invece di stringhe JSON.

4. IsValidJson

Scopo: Verifica la validità di una stringa JSON

Nome Tipo Direzione
json String Input
errorMessage String Output
Return Value Boolean Output

Sistema di Filtri

🎉 NUOVO: Sintassi Semplificata v1.1!

Ora puoi usare una sintassi super semplice senza escape di virgolette!

Formato Semplificato (RACCOMANDATO) ✅ 🆕 Nuovo

Usa il parametro filtersString con questa sintassi:

PropertyName Operator Value // Filtro singolo "Occurred Equals true" // Filtri multipli (separati da punto e virgola) "Occurred Equals true; Code GreaterThan 0" // Con stringhe (usa apici se contiene spazi) "Msg NotEquals ''" "Status Contains 'Error'" "Name StartsWith Test"

Valori Supportati

Tipo Esempi Note
Boolean true, false Case insensitive
Numeri 0, 10, 3.14, -5 Int o double
Stringhe Test, 'Hello World', "Error" Apici opzionali (necessari se contiene spazi)
Null null Case insensitive

Operatori Disponibili

Operatore Tipo Dati Descrizione Esempio
Equals Tutti Uguale a Status Equals Active
NotEquals Tutti Diverso da Code NotEquals 0
GreaterThan Numeri Maggiore di Count GreaterThan 0
LessThan Numeri Minore di Value LessThan 100
GreaterThanOrEqual Numeri Maggiore o uguale Score GreaterThanOrEqual 50
LessThanOrEqual Numeri Minore o uguale Temp LessThanOrEqual 80
Contains Stringhe Contiene Message Contains Error
StartsWith Stringhe Inizia con Name StartsWith Test
EndsWith Stringhe Termina con File EndsWith .txt
⚠️ Formato JSON Vecchio (Ancora Supportato):
Il parametro filtersJson funziona ancora per retrocompatibilità, ma in TestStand richiede escape:
"[{\"Property\": \"Occurred\", \"Operator\": \"Equals\", \"Value\": true}]"
→ Usa filtersString invece!
🎯 Confronto Sintassi
Prima (filtersJson) Adesso (filtersString) 🆕
"[{\"Property\": \"Occurred\", \"Operator\": \"Equals\", \"Value\": true}]" "Occurred Equals true"
❌ Complicato, errori facili ✅ Semplice, leggibile

💡 Esempi Pratici

Esempio 1: Rimuovere Oggetti Vuoti

Caso d'uso: Pulizia di oggetti "Error" vuoti dopo test superati

Input JSON:

{ "Test1": { "Error": {}, "Result": "PASS" }, "Test2": { "Error": {}, "Result": "PASS" } }

Metodo: RemoveJsonElementFromString

Parametro Valore
targetToRemove {}

Output:

{ "Test1": { "Result": "PASS" }, "Test2": { "Result": "PASS" } } // removedCount = 2

Esempio 2: Rimuovere Proprietà per Nome (Senza Filtri)

Caso d'uso: Eliminare tutte le proprietà "Common" dal JSON

Input JSON:

{ "Common": {"ver": "1.0"}, "Data": { "Common": {"ver": "1.0"}, "Values": [1, 2, 3] } }

Metodo: RemovePropertyByName

Parametro Valore
propertyName "Common"
filtersString "" (vuoto - rimuovi tutto)

Output:

{ "Data": { "Values": [1, 2, 3] } } // removedCount = 2, skippedCount = 0

Esempio 3: Rimuovere con Filtro Booleano 🆕 Nuova Sintassi

Caso d'uso: Rimuovere "Error" SOLO se Occurred = false

Input JSON:

{ "Test1": { "Error": { "Code": 0, "Msg": "", "Occurred": false } }, "Test2": { "Error": { "Code": 5, "Msg": "Timeout", "Occurred": true } } }

Metodo: RemovePropertyByName

Parametro Valore
propertyName "Error"
filtersString 🆕 "Occurred Equals true"
✅ Zero escape! Zero virgolette complicate!
In TestStand scrivi semplicemente: Locals.FiltersString = "Occurred Equals true"
ℹ️ Logica Filtro: Se il filtro matcha, la proprietà viene SKIPPATA (non rimossa).
In questo caso: rimuovi "Error" dove Occurred ≠ true

Output:

{ "Test1": {}, "Test2": { "Error": { "Code": 5, "Msg": "Timeout", "Occurred": true } } } // removedCount = 1, skippedCount = 1

Esempio 4: Filtro Numerico 🆕

Caso d'uso: Rimuovere "Results" solo se Count = 0

Parametri:

Parametro Valore
propertyName "Results"
filtersString "Count GreaterThan 0"

Esempio 5: Filtro Stringa 🆕

Caso d'uso: Rimuovere "Status" solo se Msg è vuota

Parametri:

Parametro Valore
propertyName "Status"
filtersString "Msg NotEquals ''"

Nota: Gli apici '' sono necessari per stringhe vuote.

Esempio 6: Rimuovere Array Completi

Caso d'uso: Rimuovere array specifici in base al contenuto

Metodo: RemoveJsonElementFromString

Parametro Valore
targetToRemove [1, 2, 3]

Esempio 7: Filtri Multipli 🆕 Nuovo

Caso d'uso: Rimuovere "Data" con logica complessa (Count = 0 OPPURE Status = "Empty")

Parametri:

Parametro Valore
propertyName "Data"
filtersString "Count Equals 0; Status Equals Empty"
ℹ️ Logica OR: Viene skippato se almeno uno dei filtri matcha.

Esempio 8: Operatori Stringhe Avanzati 🆕

Caso d'uso: Rimuovere "Log" solo se non contiene "Error"

Esempi operatori:

// Inizia con "Name StartsWith Test" // Termina con "FileName EndsWith .log" // Contiene "Path Contains Windows" // Con spazi (usa apici) "Message Contains 'Fatal Error'"

🔌 Integrazione con TestStand

Step 1: Configurazione Base

  1. Apri TestStand Sequence Editor
  2. Inserisci un nuovo step
  3. Adapter Type: .NET Adapter
  4. Tab .NET → Use Assembly
  5. Assembly Path: C:\TestStand\Components\JsonCleaner\JsonCleaner.dll
  6. Class Name: JsonCleaner.JsonCleanerTool
  7. Method: Seleziona il metodo desiderato

Step 2: Configurazione Parametri

Nel tab Parameters, mappa i parametri del metodo alle variabili TestStand:

Esempio: RemovePropertyByName (CON NUOVA SINTASSI) 🆕

Parameter Direction TestStand Variable
inputJson Input Locals.MyJsonString
propertyName Input "Error"
outputJson Output Locals.CleanedJson
removedCount Output Locals.RemovedCount
skippedCount Output Locals.SkippedCount
logOutput Output Locals.LogOutput
filtersJson Input "" (lascia vuoto)
filtersString 🆕 Input Locals.FiltersString
enableLogging Input True
Return Value Output Locals.Success

Step 3: Preparazione Variabili

// Statement Step - Prepara JSON input Locals.MyJsonString = FileGlobals.TestResults.AsPropertyObject().AsJSON(True) // Statement Step - NUOVA SINTASSI SEMPLIFICATA! 🆕 Locals.FiltersString = "Occurred Equals true" // Oppure con filtri multipli Locals.FiltersString = "Occurred Equals true; Code GreaterThan 0" // .NET Step - Chiama JsonCleaner // Statement Step - Verifica risultato If (Locals.Success == True) Locals.CleanedObject = JSON.Parse(Locals.CleanedJson) Log("Rimosse: " + Locals.RemovedCount) Log("Skippate: " + Locals.SkippedCount) EndIf

Best Practices TestStand

✅ Raccomandazioni:
  • Usa Create New Instance (default) - non serve Dispose
  • Abilita logging solo in fase di debug
  • Verifica sempre Return Value prima di usare outputJson
  • Controlla logOutput in caso di errori
  • Per JSON grandi (>10MB), considera di processarli a blocchi
  • 🆕 Usa sempre filtersString invece di filtersJson

🔍 Troubleshooting

Problema: TestStand non trova la DLL

Errore: "Assembly not found" o "Type not found"

Soluzioni:
  • Verifica che JsonCleaner.dll e Newtonsoft.Json.dll siano nella stessa cartella
  • Controlla la versione .NET (Framework 4.8 vs .NET 6+)
  • Se usi .NET 6+, verifica che TestStand sia 2021 o superiore
  • Prova a mettere il path assoluto

Problema: Filtri non funzionano 🆕

Verifica:
  • La sintassi filtersString è corretta? Format: "PropertyName Operator Value"
  • Il nome della proprietà è corretto (case-sensitive)?
  • Gli operatori sono scritti corretti? Es: Equals non Equal
  • Se il valore è una stringa con spazi, hai usato gli apici?
  • Per filtri multipli, hai separato con ;?
  • Abilita logging e controlla logOutput
